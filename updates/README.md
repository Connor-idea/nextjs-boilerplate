# Update Notes

Store one Markdown file per meaningful UI or behavior change.

## Naming
- Use `YYYY-MM-DD-short-topic.md`
- Example: `2026-05-07-tailwind-table-sync.md`

## Required sections
- Summary
- Scope
- Files
- Validation
- Risks or follow-up

## Rules
- Record what changed, why it changed, and how it was verified.
- Keep entries factual and concise.
- If a change updates shared UI abstractions, mention the shared class/helper/token that now owns the behavior.
